## Problem Statement

> *"Build an AI-powered productivity companion that proactively assists users in planning, prioritizing, and completing tasks before deadlines are missed. The solution should move beyond traditional reminders and focus on helping users take meaningful action."*

**Why traditional apps fail:**
- Passive to-do apps store tasks but never help you *execute* them
- Reminders are ignored — people snooze and return to the same chaos
- Calendar apps show what's scheduled, not what matters *right now*
- No tool connects the dots between tasks, calendar, habits, and goals in real-time

---

## Our Solution

**ResQ** is a full-stack AI productivity companion powered by **Google Gemini** that runs a continuous intelligence loop — analyzing your entire workload every 5 minutes and taking *action before deadlines are missed*.

```
Every 5 minutes → Cron scans all users:
  ├── Tasks due within 2 hours      → Deadline alert (push + email + voice)
  ├── Meetings starting in 15 min   → Pre-meeting alert
  ├── Habits unfinished at 8 PM     → Habit reminder
  ├── Goals < 20% with deadline     → Goal pace alert
  └── Morning briefing after 6 AM  → Gemini-generated spoken briefing
```

You don't open ResQ to stay productive — **ResQ comes to you**.

## Judging Criteria Alignment

### Problem Solving & Impact (20%)

ResQ directly solves the hackathon problem through:
- **Proactive scanning** every 5 minutes across all users (`cronService.js`)
- **Unified intelligence** — single Gemini call analyzes all productivity modules (`generateGlobalPriority`)
- **Actual action execution** — not suggestions, but real DB writes (`voiceIntentService.js`, 1,367 lines)
- **Multi-channel delivery** — Socket.IO + WebPush + Email + Voice simultaneously

### Agentic Depth (20%)

**Perception:** Continuous microphone monitoring, cron-based data scanning, real-time socket events

**Reasoning (Google Gemini):**
- `generateGlobalPriority()` — cross-domain ranking
- `handleVoiceCommand()` — 15+ intent types with action payloads
- `generateAutoSchedule()` — conflict-aware scheduling
- `inferTaskUrgency()` — semantic urgency from task semantics

**Action Execution (`voiceIntentService.js`):**
- `create_task` → MongoDB write
- `schedule_event` → CalendarEvent document created
- `reschedule_event` → conflict resolution + update
- `complete_task` / `delete_task` / `update_task_priority`
- `start_focus` / `stop_focus` → UI state mutation
- `complete_habit` → habit completion record
- `navigate_tab` → UI navigation control
- `change_theme` → DB preference update + live UI switch

**Proactive Initiation:** ResQ speaks to users without being asked (`setPendingProactiveCommand`)

## AI Architecture

### All 10 Gemini Pipelines

| Function | Input | Output | User Benefit |
|---------|-------|--------|-------------|
| `generateDailySummary` | User + tasks + events + habits + goals | 2–3 sentence briefing | Spoken morning brief |
| `generateTaskPriority` | Task array | `[{id, reason}]` ranked | Dashboard priority |
| `generateAutoSchedule` | Tasks + events + hours | `[{taskId, day, timeSlot}]` | Auto-filled calendar |
| `generateHabitInsight` | Habit + 30-day history | `{insight, tip}` | Habit coaching |
| `generateGoalBreakdown` | Goal + target date | `[{week, milestone, effort}]` | Milestone plan |
| `cleanManualMilestones` | Raw text milestones | Cleaned milestone JSON | Better goal plans |
| `handleVoiceCommand` | Transcript + context | `{intent, response, action}` | Voice execution |
| `generateNotificationMessage` | Type + context | Short push message | Personalized alerts |
| `generateGlobalPriority` | All modules | Top 4 `[{id, type, score, reason}]` | Unified priority |
| `inferTaskUrgency` | Title + due date | Integer 1–10 | Auto urgency |

### Architecture Flow

```
User Input (Voice/UI)
         ↓
   Authentication (JWT)
         ↓
   Express Controller
         ↓
   Google Gemini AI
         ↓
   Action Execution
   (MongoDB writes)
         ↓
   Response Delivery
   (Socket.IO + Push + Email + Voice)
```

## Database Schema

### Collections
- **User** — auth, subscription, settings, Google tokens, VAPID subscriptions
- **Task** — title, urgency (1–10), dueDate, estimatedMinutes, aiPriorityRank
- **CalendarEvent** — startTime, endTime, type, googleEventId, taskId, aiGenerated
- **Habit** — name, targetDays, streak, completions[]
- **Goal** — title, targetDate, progress%, milestones[], keyResults[]
- **Notification** — type, message, read status
- **Payment** — Razorpay IDs, amount, status, plan_type, subscription dates
- **TrialTracking** — email + phone uniqueness for trial abuse prevention

### Key Relationships
- Task → CalendarEvent (1:many, via taskId — AI focus blocks)
- User → all collections (1:many)
- Payment → User subscription activation

## Installation

### Prerequisites
- Node.js 18+, MongoDB Atlas, Google Cloud Console project

### Setup

```bash
git clone https://github.com/its5zoo/ResQ.git
cd ResQ

# Backend
cd backend && npm install

# Frontend
cd ../frontend && npm install
```

### Backend `.env`
```env
PORT=5000
MONGO_URI=mongodb+srv://...
JWT_SECRET=your_jwt_secret
GEMINI_API_KEY=your_gemini_api_key
GEMINI_MODEL=gemini-3.1-flash-lite
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
GOOGLE_REDIRECT_URI=http://localhost:5000/api/google/callback
ELEVENLABS_API_KEY=your_elevenlabs_key
VAPID_PUBLIC_KEY=your_vapid_public_key
VAPID_PRIVATE_KEY=your_vapid_private_key
VAPID_EMAIL=mailto:your@email.com
RAZORPAY_KEY_ID=rzp_test_...
RAZORPAY_KEY_SECRET=your_razorpay_secret
CLIENT_URL=http://localhost:5173
```

### Frontend `.env`
```env
VITE_API_URL=http://localhost:5000/api
VITE_SOCKET_URL=http://localhost:5000
VITE_RAZORPAY_KEY_ID=rzp_test_...
```

### Generate VAPID Keys
```bash
cd backend && npx web-push generate-vapid-keys
```

### Run
```bash
# Backend
cd backend && npm run dev

# Frontend (new terminal)
cd frontend && npm run dev
```

App runs at `http://localhost:5173`
